# Автоматизированная система сортировки писем с корпоративной почты

Система для отдела IT-поддержки, которая читает входящие обращения из папки, классифицирует их по категориям, распределяет по папкам и формирует отчёт о проделанной работе.

## Возможности

- Чтение писем в форматах TXT и JSON
- Классификация писем по триггерным словам
- Распределение писем по категориям
- Обработка ошибок чтения
- Ведение логов
- Группировка инцидентов по системам
- Формирование отчета по инцидентам (в текстовом и графическом формате)

## Категории

Письма классифицируются по триггерным словам в теме и теле письма. Слово в теме имеет коэффициент 2, а слово в теле — 1.

Категории:

- Инциденты
- Техподдержка
- Оборудование
- Доступы
- Финансы и документы
- График работы
- Спам
- Встречи
- Новостная сводка
- Мониторинг

Если письмо не подходит ни под одну из категорий, оно перемещается в папку "Неизвестное".
Если письмо набирает одинаковое количество триггерных слов по разным категориям, оно попадает в папку "Смешанная категория".
Такие категории выбраны после анализа входящих писем. В основу логики легла не только тема письма, а то, какая реакция требуется от IT-поддержки: запрос доступа — Доступы, неисправность принтера — Оборудование, критический сбой — Инциденты.

## Расширение: группировка инцидентов

Поверх базовой классификации система дополнительно обрабатывает письма категории Инциденты. Один сбой обычно порождает несколько похожих обращений от разных сотрудников, поэтому расширение объединяет такие письма в один инцидент. Это помогает видеть реальный масштаб проблемы, а не считать похожие обращения отдельно.
После распределения писем система анализирует папку `result/Инциденты`. Для объединения обращений анализируются упомянутые в письмах системы: например, корпоративный портал, BI-система, GitLab, Confluence, Slack, Service Desk и другие. Письма с совпадающими системами считаются относящимися к одному инциденту.
Для каждого инцидента считается количество обращений и количество уникальных отправителей, то есть сколько человек реально столкнулось с проблемой.
Группировка выполняется именно по системе, потому что сотрудники могут описывать одну и ту же проблему разными словами. При этом одинаковая тема письма не всегда означает один и тот же сбой, поэтому важнее определить, какая система затронута.
Результат расширения:

- `result/incidents.md` — отчёт по инцидентам, отсортированный по числу обращений;
- `result/incidents.png` — диаграмма распределения инцидентов по системам;
- в консоль выводится приоритетный инцидент с наибольшим числом обращений.
Расширение реализовано в `src/merge_incidents.py` и использует библиотеку matplotlib.

## Структура проекта

- `src/reader.py` — чтение писем в форматах txt и json, обработка ошибок чтения;
- `src/classifier.py` — классификация писем по триггерным словам;
- `src/distributor.py` — перемещение писем по папкам категорий;
- `src/merge_incidents.py` — группировка инцидентов по системам;
- `src/main.py` — основной файл, связывающий остальные части программы;
- `tests/` — тесты чтения и классификации;
- `run.sh` — скрипт запуска;
- `reset.sh` — сброс проекта в исходное состояние для повторного запуска;
- `inbox/` — папка с входящими письмами;
- `inbox_backup/` — эталонная копия писем;
- `result/` — результат сортировки.

## Результат

После запуска программа:

- раскладывает письма по папкам `result/Категория`;
- перемещает письма, которые не удалось прочитать, в `result/errors`;
- записывает ход работы и ошибки в файл `logs.txt`;
- дополнительно группирует инциденты;
- сохраняет отчёт по инцидентам в `result/incidents.md`;
- сохраняет диаграмму по инцидентам в `result/incidents.png`;
- выводит в терминал статистику: сколько всего писем, сколько обработано, сколько с ошибками;
- выводит приоритетный инцидент и общее число инцидентов.

## Запуск

Для построения графика нужна библиотека matplotlib:
```bash
pip install matplotlib
```

Папка `inbox/` не хранится в репозитории, поэтому перед первым запуском её нужно подготовить из `inbox_backup/` командой:
```bash
bash reset.sh
```

После этого запуск:
```bash
bash run.sh
```

`reset.sh` также используется перед повторными запусками — он очищает `result/` и заново наполняет `inbox/` из `inbox_backup/`.

## Тесты

Реализованы с помощью библиотеки `pytest`. Тесты не зависят от папки `inbox/`, потому что используют собственные входные данные.
Покрыты: чтение писем (txt, json, пустой и битый файл), классификация (включая "Неизвестное" и "Смешанная категория"), распределение писем по папкам и группировка инцидентов (включая неизвестный инцидент и отсутствие папки). Запуск тестов:
```bash
pytest
```
